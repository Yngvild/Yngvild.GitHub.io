Demography and matrix models

4 Beyond age structure

Learning goals

  • Understand the key components of a general stage structured matrix models.

  • Mention and discuss examples of different stage structure.


4.1 Matrix models are very flexible

The life table and associated parameters defined in the previous chapters apply only to age structured populations. However, the methods you learned for matrix models can be used with any kind of projection matrix (provided it fulfills certain mathematical assumptions). Here we mention only a few common examples and look at how the projection matrix changes. You can apply the exact same methods above on your own to explore the stable structure, reproductive values, etc. The only difference is that the results refer to stages instead of age classes.


4.2 Life cycle stages

Distinct life cycle stages are typical for many insects. For each stage (e.g. egg, larvae, pupae, adult), the individuals can either remain in their current stage for one more time step, or make a transition to the next stage (development). With this kind of structure, the projection matrix will includes transition probabilities \(g_j\) (probability of transition out of the stage) in addition to stage-specific survival \(s_j\) and fertility \(f_j\), and takes the form


\[\begin{align} \begin{split} \mathbf{A}=\left[ \begin{matrix} f_1+s_1(1-g_1) & f_2 & \cdots &f_{k-1}&f_k\\ s_1 g_1&s_2(1-g_2) & \cdots&0&0\\ 0 &s_2 g_2&\ddots&0&0\\ \vdots & \vdots & \ddots&s_{k-1}(1-g_{k-1})&0 \\ 0 &0 & \cdots&s_{k-1}g_{k-1}&s_k \\ \end{matrix} \right]. \end{split} \tag{4.1} \end{align}\]


Note that in the final stage (typically the adult stage), all individuals can only survive and remain. This kind of projection matrix is known as a Lefkovitch matrix, named after the work of Lefkovitch (1965).


4.2.1 Example of Lefkovitch matrix: Wild boar


Figure 4.1: Wild boar (Sus scrofa).


This example is one of many studies using a Lefkovitch matrix to separate among main life cycle stages. Such models are often used because it captures the main differences among individuals without becoming too complex. In a study on wild boar, Bieber and Ruf (2005) constructed a model with three classes: Juveniles (age 0-1), yearlings (age 1-2) and adults (age 2+).

The projection matrix for this wild boar model is given by

\[ \mathbf{A}=\left[\begin{matrix} 0.26 & 0.94 &1.93 \\ 0.33&0 &0 \\ 0&0.4 &0.66 \\ \end{matrix}\right]. \]


As you may have noticed, this matrix is quite similar to an age structured model (Leslie matrix), but with the exception that adults do not necessarily die after reaching age 2, but can continue to live so that the final stage contains all individuals of age 2 and more (2+).


4.3 Age by stage models

Individuals in a population will always differ in more than one way, and sometimes (depending on the study question as well as the species) we have to account for more than one state variable in order to adequately describe the life history. For instance, we may need to combine age and life cycle stage, if the age differences within each life cycle stage are large enough. The key is to define such a model is to first define new classes from each possible combination of age and stage. If there are \(k\) age classes, and \(m\) stages, the new age by stage model will have \(\phi=k*m\) classes in total. From this we can define a new projection matrix of dimension \(\phi \times \phi\), which can be analysed using the same methods as before to find \(\lambda\) and other parameters.


4.4 Size structure: Integral projection models

Many state variables are continuous in nature, such as body size. Matrix models can lump individuals into different discrete size classes, but this adds an extra modeling decision of where to set the limits for each size class. This can be avoided by using an integral projection model (IPM; Easterling, Ellner, and Dixon (2000)). Instead of a population vector with a discrete number of stages, these models follow a continuous trait distribution \(n(z)\), where \(z\) represents the trait measure (e.g. size), and instead of a projection matrix IPMs use a projection kernel (or surface). You can think of an IPM as a matrix model with infinitely many stages, and in fact they belong to the same class of models (matrix models). An IPM can be analysed with matrix model methods to find \(\lambda\) and other life history parameters. The main difference is in how these parameters are (typically) estimated from data. We will not go into details regarding IPMs here, but you have already learned the methods for analyzing them, as they are the same as for matrix models.
